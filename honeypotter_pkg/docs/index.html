Honeypotter

Generate non-semantic in-dist and OOD distractor classes to stress-test image classifiers.

GitHub Download v0.5.0

Why Honeypots?

Honeypot classes (e.g., stripes, checkers, blobs) have no semantic meaning—they exist only to hijack classifier confidence. To evaluate properly, append honeypot class names to your dataset’s label space and measure steal-rate.

In-Distribution vs OOD Colors

Render each family with palette (ImageNet/THINGS priors) or ood (neon/unrealistic). Below: left are palette samples, right are OOD samples.

CLI

honeypotter generate \
  --out ./honeypots \
  --n_categories 100 --per_class 100 \
  --families checker,stripes,dots,blob,solid \
  --color_mode ood --ood_palette ./ood_colors.json \
  --pattern_blur_sigma 3.0 --global_blur_sigma 0.0 \
  --seed 0
